Git - Grundlagen

Versionsverwaltung

Was ist Versionsverwaltung?
  • Protokollierung von Änderungen an Dateien über die Zeit
  • Möglichkeit zur Rückkehr zu bestimmten Versionen
  • Version Control System (VCS)
  • Centralized Version Control System (CVCS)
  • Distributed Version Control System (DVCS)

Lokale Versionsverwaltung

  • Alle Dateien in ein separates Verzeichnis kopieren
  • Flexible Versionierung ohne Serverabhängigkeit
  • Minimiertes Risiko von Datenverlust bei zentralen Serverausfällen

Probleme mit VCS

  • Verbreitet aber fehleranfällig
  • Risiko von Arbeiten im falschen Verzeichnis
  • Überschreiben von Dateien, die nicht überschrieben werden sollten
  • Zusammenarbeit auf verschiedenen Systemen

Zentrale Versionsverwaltung

  • Basiert auf einem zentralen Server
  • Transparenz über Projekttätigkeiten der Beteiligten
  • Genau definierte Administratorrechte
  • Einfachere Administration im Vergleich zu lokalen Systemen

Probleme mit CVCS

  • Risiko eines Systemausfalls bei Serverausfall
  • Keine Zusammenarbeit oder Versionierung bei Serverausfall
  • Risiko von Datenverlust bei beschädigter Serverfestplatte

Verteilte Versionsverwaltung

  • Vollständige lokale Kopie für unabhängige Arbeitsumgebung
  • Dezentrale Sicherung (Clone) als Backup
  • Unabhängigkeit von Serverzustand ermöglicht offline zu arbeiten

Probleme mit DVCS

  • Komplexität für Anfänger erschwert die Einarbeitung in die Konzepte von DVCS.
  • Potenziell größere Repositorygrößen bei vielen erstellten Klones.
  • Notwendigkeit der Versionskontrolle, um sicherzustellen, dass alle Änderungen eingespielt werden.
  • Komplexe Arbeitsabläufe erfordern Erfahrung und können für unerfahrene Nutzer überwältigend sein.

Versionskontrolle mit Git

  • Verteiltes Versionsverwaltungssystem
  • Entwickelt von Linus Thorvalds im Jahr 2005
  • Unter der GNU-GPLv2-Lizenz veröffentlicht

Warum Versionskontrolle wichtig ist?

  • Effiziente Zusammenarbeit: Ermöglicht mehreren Personen, gleichzeitig an einem Projekt zu arbeiten.
  • Rückverfolgbarkeit: Verfolgt Änderungen und ermöglicht die Rückkehr zu früheren Versionen.
  • Fehlerbehebung: Vereinfacht das Auffinden und Beheben von Fehlern im Code.
  • Dokumentation: Protokolliert den Verlauf des Projekts und dient als Dokumentation.
  • Teamarbeit: Unterstützt die Koordination von Teammitgliedern und deren Beiträgen.
